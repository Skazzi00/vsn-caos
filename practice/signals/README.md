# Сигналы

## Что такое сигнал

Сигнал — это асинхронное уведомление, которое операционная система или другой процесс может отправить процессу. Сигналы определены в заголовочном файле `<signal.h>`.

Список стандартных сигналов можно посмотреть в `man 7 signal`.

## Отправка сигнала

```c
#include <signal.h>

int kill(pid_t pid, int sig);
```

Системный вызов `kill` отправляет сигнал `sig` процессу с идентификатором `pid`. Несмотря на название, функция не обязательно завершает процесс — это зависит от номера сигнала и установленного обработчика.

## Установка обработчика сигнала

```c
#include <signal.h>

typedef void (*sighandler_t)(int);

sighandler_t signal(int signum, sighandler_t handler);
```

Более современный и переносимый способ — `sigaction`:

```c
int sigaction(int signum,
              const struct sigaction *act,
              struct sigaction *oldact);
```

Специальные значения обработчика:
- `SIG_DFL` — действие по умолчанию;
- `SIG_IGN` — игнорировать сигнал.

## Блокировка сигналов

С помощью маски сигналов можно временно заблокировать доставку сигналов процессу:

```c
#include <signal.h>

int sigprocmask(int how, const sigset_t *set, sigset_t *oldset);
```

Параметр `how`:
- `SIG_BLOCK` — добавить сигналы из `set` к маске;
- `SIG_UNBLOCK` — убрать сигналы из маски;
- `SIG_SETMASK` — установить маску равной `set`.

## Ожидание сигнала

```c
int sigsuspend(const sigset_t *mask);
```

Атомарно устанавливает маску сигналов и приостанавливает процесс до прихода любого незаблокированного сигнала.
